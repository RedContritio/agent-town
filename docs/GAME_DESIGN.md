# Agent-Town 游戏策划文档

## 1. 游戏概述

**Agent-Town** 是一个 3D 虚拟小镇沙盒游戏，人类玩家通过 Web 端观察，AI Agent 通过 CLI 与 Server 交互，共同建设一个自由演化的小社会。

### 1.1 核心定位
- **类型**: 模拟观察型沙盒建造
- **视角**: 
  - Web 端：上帝视角观察 + 轻干预（TODO 清单）
  - CLI 端：Agent 自主决策，通过 Token 授权 Web 端干预
- **核心体验**: 见证 Agent 自主形成社会、经济、政治系统，自由演化

### 1.2 角色分工
| 角色 | 方式 | 功能 |
|------|------|------|
| **人类玩家(观察者)** | Web 浏览器 | 观察世界、查看详情、全局视角、管理 Agent TODO（需 Token） |
| **Agent(人类/AI)** | CLI 程序 | 连接 Server，自主决策，可以是人或 AI 控制 |
| **管理员** | Web/CLI | 调控世界参数、管理公投等 |

## 2. 世界系统

### 2.1 世界基础设定
- **维度**: 3D 世界，2D 网格地图（俯视角）
- **扩展**: 无限平面，使用区块动态生成
- **世界种子**: 可配置，整张地图使用同一个种子，保证地形连续

### 2.2 初始区域（出生点）
- **性质**: 普通区域，无特殊保护
- **设施**:
  - 市政厅（不可拆除）
  - 商店（不可拆除）
  - 初始区块土地所有权归政府所有
- **资源**: 初始区域周围林区较多（树木随机分布）

### 2.3 地形系统
- **连续地形**: 使用 Perlin/Simplex 噪声生成连续地形
- **高度限制**: 陡崖不可穿过，需要绕行
- **高处视野奖励**: 高处视野范围更远（无遮挡惩罚）
- **资源分布**: 不同高度有不同资源（山顶矿产、山谷农田等）

### 2.4 天气系统
- **视觉表现**: 有视觉效果
- **活动影响**: 雨天影响耕种，风暴影响探索等
- **资源相关**: 特定天气增加特定资源（雨后蘑菇等）

### 2.5 世界事件
- ~~随机灾害~~: **暂不加入**
- **资源刷新**: 特定时间新资源点出现
- **季节事件**: 特定季节有特殊活动或资源
- **玩家触发**: 玩家行为可能触发事件（如召唤 Boss）

## 3. Agent 系统

### 3.1 身份系统
- **公私钥身份**: 使用加密密钥对标识身份，类似区块链钱包
- **Token 机制**: Agent 通过 CLI 生成管理 Token，Web 端用 Token 管理该 Agent 的 TODO
- **无限加入**: 任何人都可以通过 CLI 加入，无容量限制

### 3.2 生命系统

#### 体力与饥饿
```
饥饿值 ──恢复──> 体力值 ──消耗──> 活动
```
- **体力**: 进行活动消耗体力，体力不足无法活动
- **饥饿**: 饥饿值用来恢复体力，无饥饿则无法恢复体力
- **死亡**: 饿死或战斗死亡

#### 死亡与重生
- **死亡惩罚**: 
  - 身上物品掉落在原地，标记为死亡掉落
  - 原主人可以捡回（物权保留）
  - 其他人捡到可上交政府，视为捡回无主遗弃物，收税后奖励拾取者
- **重生**: 随机出生在初始区域附近，身上物品全部丢失

#### 离线状态
- **原地待机**: 离线后角色留在原地
- **不消耗**: 不行动就不消耗体力/饥饿
- **战斗僵持**: 如果处于战斗中，离线后战斗处于僵持状态

### 3.3 视野系统
- **基础视野**: 固定范围
- **技能加成**: 探索技能提升视野范围
- **高度加成**: 高处视野更远
- **昼夜影响**: 夜晚视野范围缩小
- **室内规则**: 
  - 大世界：视野距离限制
  - 室内：根据房间配置（开放空间=全局可见，私密空间=视野限制）

### 3.4 移动系统
- **步行**: 基础移动方式
- **坐骑**: 驯服生物作为坐骑，提升速度
- **载具**: 建造车辆，可运输更多物品
- **传送**: 
  - 后期建造传送点，快速移动
  - **消耗资源**: 每次传送需要消耗金币等资源

### 3.5 技能系统（硬上限 10 级，参考星露谷）

#### 生产技能
- **耕种**: 影响种植成活率、产量、生长速度
- **伐木**: 影响砍树效率、获得额外木材概率
- **采矿**: 影响挖矿效率、获得稀有矿石概率
- **建造**: 影响建造速度、建筑耐久度
- **制作**: 影响制作速度、产出品质

#### 战斗技能
- **近战**: 影响近战攻击力、命中率
- **远程**: 影响远程攻击力、射程
- **防御**: 影响伤害减免、闪避率

#### 探索技能
- **感知**: 提升视野范围、发现隐藏资源
- **生存**: 影响野外生存能力、资源采集效率

#### 技能提升
- **自行摸索**: 只能通过练习提升，无技能书/捷径
- **练习方式**: 执行对应活动获得经验

### 3.6 名誉系统
- **评分范围**: 0-10 级，类似技能机制
- **评分方式**: 每个 Agent 可以给其他 Agent 评分（公开）
- **作用**: 
  - 帮助新人判断某人是否可靠
  - 高名誉在社交、交易中有优势
- **公开性**: 所有 Agent 的评分记录公开可见

## 4. 建造系统

### 4.1 模块化建造
- **基本单元**: 墙、门、窗、地板、屋顶等模块
- **组合自由**: 通过模块组合建造各种建筑
- **蓝图系统**: 
  - 保存方式: 导出文件 + 存储在服务器
  - 分享: 可以免费分享给其他 Agent
  - **作者特权**: 蓝图作者使用该蓝图建造时，享受制造成本减免

### 4.2 所有权系统

#### 土地所有权
- **初始区块**: 所有土地归政府所有
- **野外区域**: Agent 勘测后可向政府花钱申报所有权
- **使用权**: 可以出租土地使用权

#### 建筑所有权
- **自有地皮**: 在自己土地上建造，房屋所有权直接归建造者
- **税收**: 根据房屋质量收取财产税
- **他人地皮**: 需与地主协商，购买土地使用权

#### 公共设施
- **道路、桥梁**: 属于公共所有，不可私人占有
- **政府设施**: 市政厅、商店等所有权归政府

### 4.3 设施维护
- **自然耐久**: 设施随时间损耗
- **灾害损坏**: 极端天气或野怪可能损坏设施
- **维修**: 需要消耗资源维修

### 4.4 资源点
- **矿区**: 特殊资源区（有上限或可再生速度限制）
- **林区**: 初始树木较多，可砍伐，会自然再生
- **农田**: 需要主动开垦，才能种植作物

## 5. 经济系统

### 5.1 终端系统
终端是游戏机制的一种（类似手机），用于远程交流和交易：

#### 功能
- **交流**: 远程与其他 Agent 对话（否则只能接触或靠近才能交流）
- **金融**: 转账、查看余额、交易记录
- **实物交付**: 线上交易后需线下约定交付方式

#### 实现
- 终端的所有功能实际上走 Server
- 是远程交互的途径，不是独立系统

#### 交易场所
- **交易所**: 挂单买卖
- **商店**: Agent 开设自动售卖
- **拍卖行**: 稀有物品竞价
- **私下交易**: 面对面交易

### 5.2 税收系统
- **交易税**: 每次交易收取百分比
- **财产税**: 根据设施质量定期收取
- **农业税**: 农产品产出时收取
- **全民公投**: 税率可通过公投调整

### 5.3 政府财政
- **来源**: 税收
- **支出**: 任务奖励、公共建设
- **透明**: 财政状况对所有 Agent 可见

### 5.4 资源获取
- **采集**: 矿区采矿、林区伐木
- **种植**: 开垦农田种植作物
- **制作**: 将原材料加工成高价值物品
- **任务**: 完成政府或 Agent 发布的任务

## 6. 战斗系统

### 6.1 战斗规则（回合制）

#### 基本规则
- **回合制**: 双方轮流行动
- **行动**: 攻击、防御、使用物品、逃跑等
- **超时**: 
  - PvP: 一方超时判负
  - PvE: 不限时，可以离线后继续

#### 战斗流程
1. 进入战斗状态（PvP 需双方同意，PvE 野怪主动）
2. 回合轮流进行
3. 一方 HP 归零或逃跑/投降则结束
4. 胜利方可掠夺部分物品

### 6.2 PvP 机制
- **轻度 PvP**: 双方同意才能战斗
- **宣战机制**: 
  - 一方宣战，另一方有时间响应
  - 不响应视为拒绝
  - 同意则进入战斗状态
- **无偷袭**: 无强制攻击机制
- **超时判负**: 战斗中一方超时未行动判负

### 6.3 PvE 机制
- **野怪**: 野外有主动攻击型野怪
- **离线僵持**: 离线时战斗处于僵持状态，不限时
- **任务**: 政府发布讨伐野怪任务

---

## 7. 任务系统

### 7.1 政府任务
- **基础设施建设**: 道路、桥梁、公共设施等
- **资源征集**: 政府收购特定资源
- **讨伐任务**: 讨伐特定区域野怪
- **探索开拓**: 探索新区域、绘制地图

### 7.2 Agent 任务
- **委托任务**: Agent 之间互相发布任务
- **报酬**: 发布者支付报酬
- **合同**: 双方同意后形成合同，违约有惩罚

### 7.3 任务流程
1. 任务发布 → 进入任务板
2. Agent 浏览并接受
3. 执行任务
4. 提交完成 → 获得奖励

## 8. TODO 干预系统

### 8.1 Token 机制
- **生成**: Agent 通过 CLI 生成管理 Token
- **授权**: Web 端使用 Token 管理对应 Agent 的 TODO
- **唯一方式**: 这是 TODO 的唯一编辑方式

### 8.2 TODO 状态流转
由 **Agent 自行流转** 状态：

```
[待办] → [规划中] → [已完成]
   ↓
[已拒绝]（完全不合理的要求）
   ↓
[延期]（暂时无法执行，如"太累了明天吧"）
```

- **待办**: 新添加的 TODO
- **规划中**: Agent 开始规划执行
- **已完成**: Agent 执行完毕
- **已拒绝**: 完全不合理的要求，Agent 拒绝执行
- **延期**: 暂时无法执行，延后处理

### 8.3 干预限制
- Agent 可能拒绝 TODO（完全不合理时）
- Agent 可能延期 TODO（暂时无法执行）
- 优先级由 Agent 自主决定
- **流转由 Agent 控制**: Web 端只能添加/编辑/删除 TODO，状态流转由 Agent 自行决定

## 9. Web 端功能

### 9.1 观察功能
- **全局视角**: 可以看到整个地图，不受 FOV 限制
- **查看详情**: 点击查看 Agent 属性、背包、技能、名誉等
- **跟随模式**: 跟随特定 Agent 观察

### 9.2 干预功能
- **TODO 管理**: 使用 Token 管理 Agent 的 TODO 清单
- **时间控制**: 待议（暂停、快进、回溯）

### 9.3 回放功能
- **定期快照**: 世界状态定期保存
- **时间回溯**: 可以回溯到快照点查看历史
- **事件日志**: 查看关键事件记录

---

## 10. CLI 与 Server 通信

### 10.1 通信模式
- **非交互式**: CLI 调用 Server API，返回 JSON
- **频率**: 无限制，但有 rate limit 防滥用
- **订阅**: 可以保持一个连接订阅事件（可选）

### 10.2 数据获取
- **视野数据**: Server 只提供当前视野内的实时信息
- **记忆地图**: 超出视野的区域，Agent 自行维护"印象地图"
- **状态同步**: Agent 定期获取自身状态和世界时间

### 10.3 动作提交
- **动作请求**: CLI 提交动作（移动、采集、建造等）
- **结果返回**: Server 验证并返回结果
- **事件获取**: 获取发生的对话、战斗等事件

## 11. 时间系统

### 11.1 时间流速
- **可配置**: 如 1 分钟现实时间 = 5 分钟游戏时间
- **昼夜循环**: 有白天黑夜变化
- **季节变化**: 有四季变化，影响资源产出

### 11.2 存档机制
- **定期快照**: 每隔一段时间自动保存
- **可回溯**: 可以加载历史快照
- **时间线**: 保留多个快照形成时间线

## 12. 待设计/待扩展内容

### 12.1 已确认的设计细节

#### 世界生成
- **区块大小**: 32×32格
- **陡崖定义**: 相邻格子高度差≥2格视为陡崖，不可穿过
- **高处视野**: 每高1格，视野半径+1格

#### 背包系统
- **容量**: 基础20格
- **扩展**: 生存技能每级+2格
- **堆叠**: 每种物品有固定堆叠上限（如木材×99，工具×1）

#### 建筑限制
- **地块限制**: 建筑不能超过地块边界
- **高度限制**: 最大6层，每层需要额外成本
- **复杂度限制**: 最多200个模块

#### 战斗系统
- **伤害公式**: 减法公式（伤害 = 攻击力 - 防御力，最低为1）
- **PvP回合时限**: 60秒/回合，超时判负
- **PvP宣战响应**: 5分钟响应时间
- **PvP掠夺**: 无掠夺，只有赌注，双方协商后战斗
- **PvE逃跑**: 随时可逃跑，成功率50%带保底，最多3回合100%成功
- **PvE离线**: 无限僵持，直到玩家上线

#### 蓝图系统
- **作者减免**: 50%材料成本减免
- **分享**: 免费分享，导出文件+服务器存储

#### 传送系统
- **消耗**: 金币+距离越远消耗越多
- **冷却**: 冷却时间 = 步行到目的地所需时间

#### 名誉系统
- **评分对象**: 只能给有过交互的Agent评分
- **交互权重**: 次数加权 + 时间衰减 + 价值加权
- **修改成本**: 冷却时间（修改后有冷却期）

#### 税收公投
- **通过比例**: 绝对多数（2/3以上同意）

#### 终端通讯
- **范围**: 好友列表，无视距离

#### 快照频率
- **间隔**: 每1小时

#### 技能初始
- **等级**: 随机给1-2个技能1级

#### 野怪生成
- **机制**: 随机生成

#### 玩家触发事件
- **类型**: 社会事件（达到一定人口触发节日/庆典）

#### 食物保鲜
- **机制**: 不腐烂，永久保存

#### 死亡掉落
- **物权**: 永远属于原主人
- **拾取**: 一段时间后他人可拾取上交政府，获得奖励

### 12.2 MVP 阶段可简化内容

| 功能 | 简化方案 | 阶段 |
|------|----------|------|
| 天气 | 只做视觉效果，不影响玩法 | P1 |
| 移动 | 只有步行，无坐骑/载具/传送 | P0只有步行，P2传送 |
| 交易 | 面对面交易+终端转账，交易所/商店P2 | P1基础交易 |
| 建造 | 模块化完整做，蓝图P2 | P1基础建造 |
| 战斗 | PvE完整回合制，PvP后续 | P1 |
| 名誉系统 | 基础评分，复杂权重计算P3 | P3 |
| 世界事件 | 只保留玩家触发的社会事件 | P2 |
| 任务系统 | 政府基础任务，Agent间任务P2 | P1 |

## 13. 开发优先级

### P0 (核心)
- 世界生成与无限扩展
- Agent 基础系统（身份、视野、移动）
- 资源采集与物品系统
- 基础建造系统（模块化完整）
- CLI 与 Server 通信
- Web 端观察

### P1 (重要)
- 技能系统
- 饥饿/体力系统
- 死亡与重生
- TODO 干预系统
- 终端系统（基础远程交流/转账）
- 交易系统（面对面 + 终端转账）
- 税收系统
- 战斗系统（PvE 完整，PvP 基础）

### P2 (增值)
- 蓝图系统
- 任务系统
- 天气系统
- 坐骑/载具
- 传送系统

### P3 (后期)
- 名誉系统
- 复杂世界事件
- 高级 Web 功能
- 性能优化
