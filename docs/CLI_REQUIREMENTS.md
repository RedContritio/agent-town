# at-cli 核心需求

## 1. 认证
- 本地维护私钥
- 与 server 交互生成细粒度 token
- 使用 token 进行交互

## 2. 任务执行
- 任务栈采用 LIFO 结构
- 任务状态包括：待办、规划中、已完成、已拒绝、延期
- 状态流转由 Agent 本地决策驱动

## 3. 交互
- 可向他人发起请求（交易、挑战等）
- 对方必须显式响应（接受或拒绝）
- 请求有过期时间
- 请求发出后不可由发起方撤回

## 4. 空间移动
- 移动请求指定绝对坐标
- 仅允许请求移动至已探索区域或相邻未探索格
- 移动消耗体力

## 5. 产权与经济
- 资产操作请求携带完整权属上下文
- 使用他人资源/设施时自动附加支付指令
- 产权转移时依附关系自动继承

## 6. 资源与制造
- 资源操作请求包含位置与类型
- 制造消耗资源与时间
- 成品所有权独立于原材料来源
- 建筑维护费用由当前土地主承担

## 7. 信息获取
- 仅依据 Server 返回的视野内数据做决策
- 可缓存历史感知数据（“印象地图”）
- 以 Server 实时响应为决策依据

## 8. 可靠性
- 每个操作生成唯一标识符
- 以 Server 响应为唯一可信状态源
