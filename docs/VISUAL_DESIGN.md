# Agent Town 视觉规范

约定**看起来应该像什么**，方便协作对齐；具体色值见文末附录（与 `css/style.css`、`js/renderer.js` 同步维护）。

---

## 1. 总体呈现风格

- **产品气质**：深色「观察台 / 控制台」——冷静、信息优先，不是明亮休闲手游风。界面是叠在实时世界上的**半透明 HUD**，不抢画布主体。
- **情绪与对比**：大面积暗底衬托中间高亮游戏画面；文字以浅灰为主、层级靠明度拉开；**只有状态与交互**用饱和色（青＝主操作与焦点，橙＝资源/价格/警告，绿红＝涨跌与成败）。
- **材质感**：登录与浮层常用**轻毛玻璃**（半透明 + blur），卡片与列表是**低对比描边 + 微弱底**，避免厚重实体色块铺满。
- **形状**：全局**中小圆角**（输入/按钮约 4–8px，大卡片 10–16px）；重要屏幕提示可用**大圆角药丸**（如底部进入建筑提示）。
- **动效**：**短、功能向**（约 0.15–0.3s）：hover 提亮、主按钮轻微上浮+光晕；需要用户注意时用**轻脉冲**（如断线、可进入建筑），忌夸张动画。

---

## 2. 主界面 UI 风格

- **布局**：全屏画布；信息以**可拖动浮窗**、**右侧快捷键条**、**角标式聊天条**出现，保持「世界在中间、工具在边缘」。
- **层次**：顶层固定提示（断线等）> 浮窗 > 快捷键条 > 画布。浮窗标题条用**淡青底 + 细分割线**，与正文区区分。
- **控件**：默认按钮偏**幽灵/浅底**；主行动用**实色或青渐变**；危险操作用红。列表行是**窄条 + 弱底**，选中/hover 轻微提亮或青 tint。
- **字体**：UI 用无衬线（Segoe / 微软雅黑）；**日志、代码、数字行情**用等宽，强化「终端感」。
- **密度**：侧栏与浮窗内偏**紧凑**（12–14px 级正文），避免大屏留白过多像营销站。

---

## 3. 地图与 3D 世界风格

- **视角**：**2.5D 等距**（自南偏上俯视），带轻透视；远景用**雾**与背景色融为一体，避免地平线「穿帮」，整体偏**沙盘/模型感**。
- **地形**：瓦片为**扁平色块 + 高度差**（水略低、桥略高等），颜色偏**自然写实但饱和度适中**（绿野、暖土路、清澈蓝水、金褐农田），与 UI 冷青形成**世界里温暖、界面里冷静**的对比。
- **室内**：地板木色、墙米白、门窗木与淡蓝玻璃，保持**可读、区分功能**，不追求写实纹理堆叠。

---

## 4. 建筑呈现风格

- **形式**：地图上建筑以**简易几何体块 + 类型色**区分，上方可配 **emoji/标签**（具体以当前实现为准）；**抽象示意**，不与真实比例或精细模型抢戏。
- **类型气质（协作时按此对齐新增建筑）**：

| 类型 | 呈现意图 |
|------|----------|
| shop | 暖色、亲和零售感 |
| bank | 偏冷蓝、稳重金融感 |
| exchange | 清爽绿、交易/流通感 |
| park | 自然绿、开放空间 |
| office | 中性灰蓝、办公感 |
| cafe | 暖褐、休闲餐饮 |
| home | 暖木/居住感 |

- **与地形关系**：建筑落在大块**灰地基**瓦片上时，体块色应盖过地基灰，避免「一片灰」；新增建筑类型应**先定气质与色相区间**，再落具体 Hex（见附录 B）。

---

## 5. 工具页（非主站）

独立工具页（如 tilemap_tool）当前为**另一套强调气质**（偏金褐标题与按钮），与观察台**未统一**。若合并体验，应回到第 1–2 节的 HUD 暗底 + 青强调。

## 附录 A：主色角色（简表）

| 角色 | 典型用途 |
|------|----------|
| 深青蓝底 | 页面与场景背景统一 |
| 青 | 主强调、Tab 激活、主按钮、进度、可交互名字 |
| 橙 | 价格、采矿、天气、建造消耗等「资源/系统提示」 |
| 绿 / 红 | 成功与涨 / 危险与跌 |
| 金 | 房产证等特殊高价值条目 |
